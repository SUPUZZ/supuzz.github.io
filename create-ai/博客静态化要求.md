# Supuzz博客板块静态化要求文档
## 一、核心目标
将博客预览区（id为`home-blog-grid`的容器）原本通过`fetch('blog.json')`动态加载的**前6篇博客内容**改为**静态HTML编写**，保持原有样式、交互结构、视觉效果完全不变，同时保留博客板块的标题、“View All Articles”按钮等原有元素。

## 二、静态化范围与规则
1. **仅静态化前6篇博客**：只处理博客JSON数据中排序最前的6条内容，不新增、不删减数量，与原动态加载的数量保持一致。
2. **保留原有容器结构**：
   - 保留博客板块的外层容器（`div.container#journal`、`div.blog-section`）。
   - 保留标题`<h2>Latest from the Journal</h2>`。
   - 保留底部的“View All Articles”按钮及容器（`div.btn-container`）。
   - 移除原有的“Loading updates...”加载提示文本（因为静态化后无需加载）。
3. **复用原有样式类名**：静态化的博客卡片必须使用原动态生成的类名，包括：
   - 博客卡片的外层链接标签：`class="blog-card"`。
   - 博客内容容器：`class="blog-content"`。
   - 阅读链接标签：`class="read-link"`。
   - 禁止新增或修改样式类名，确保原`style.css`中的样式能正常生效。

## 三、博客卡片静态化结构要求（与原动态代码完全一致）
### 1. 外层结构
博客卡片的外层必须是`<a>`标签（链接到博客详情页），且包含以下属性：
```html
<a href="博客详情页链接" class="blog-card" style="text-decoration: none;">
  <!-- 卡片内容 -->
</a>
```

### 2. 卡片内部结构（逐行匹配原动态代码）
```html
<!-- 博客图片容器：保留原内联样式、img标签属性 -->
<div style="height: 180px; overflow:hidden; background:#f5f5f5;">
  <img src="博客图片路径" style="object-fit:cover; height:100%; width:100%;" alt="博客标题" loading="lazy">
</div>

<!-- 博客内容容器：保留原内联样式、标签层级 -->
<div class="blog-content">
  <h3 style="font-size: 1.25rem; margin-bottom: 8px; color:#1C1B1F;">博客标题</h3>
  <p style="font-size: 0.9rem; color: #49454F; margin-bottom: 12px;">博客描述</p>
  <span class="read-link">Read Article <span class="material-symbols-rounded">arrow_forward</span></span>
</div>
```

### 3. 关键属性保留
- `img`标签：必须保留`loading="lazy"`、`alt`（值为博客标题）、内联样式`object-fit:cover; height:100%; width:100%;`。
- 所有内联样式（如`height: 180px;`、`font-size: 1.25rem;`等）必须原样保留，不得修改。
- 图标部分：保留`<span class="material-symbols-rounded">arrow_forward</span>`，与原图标样式一致。

## 四、数据填充规则
1. **使用博客JSON中的真实数据**：静态化的内容需从原`blog.json`文件中提取前6条数据，包括：
   - `href`：对应博客的`link`字段（博客详情页链接）。
   - `img src`：对应博客的`image`字段（图片路径，需保留原路径格式，如`static/blog-images/xxx.webp`）。
   - `alt`：对应博客的`title`字段。
   - `h3`内容：对应博客的`title`字段。
   - `p`内容：对应博客的`description`字段。
2. **路径格式匹配**：图片路径需与原项目中的路径格式一致（如原JSON中是`static/blog-images/`，静态化后也需保持该路径）。

## 五、JavaScript代码调整要求
1. **移除博客动态加载逻辑**：删除原代码中`fetch('blog.json')`相关的所有代码，包括：
   - `document.addEventListener('DOMContentLoaded', () => { ... })`内的博客加载代码块。
   - 相关的错误处理代码（如`console.error('加载博客失败:', ...)`）。
2. **保留其他JS逻辑**：保留移动端菜单的`toggleMenu`函数及相关逻辑，不影响头部组件的交互。

## 六、样式与兼容性要求
1. **视觉效果无变化**：静态化后的博客卡片在尺寸、间距、颜色、字体等方面，必须与原动态加载的效果完全一致。
2. **响应式兼容**：保留原有的网格布局（`class="grid"`），确保在移动端、平板、桌面端的显示效果与原动态加载一致。
3. **无新增样式**：禁止新增任何CSS样式或内联样式，仅使用原`style.css`中的样式。
